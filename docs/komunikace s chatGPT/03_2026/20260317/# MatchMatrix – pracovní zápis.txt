# 1. Cíl dne

Vyřešit **propagaci log týmů z API-Football do `public.teams.logo_url`** a ověřit, že aliasovací systém mezi různými providery funguje správně.

Současně jsme ověřovali ingest pipeline, protože některé týmy se v databázi objevily **bez loga**, přestože logo existuje v API.

---

# 2. Analýza problému

V tabulce:

```
public.teams
```

chyběly loga například pro:

* Wrexham AFC
* Chelsea FC
* Sporting Clube de Portugal

Loga jsou dostupná v:

```
staging.api_football_teams.logo
```

Ale aktualizace neprobíhala, protože:

1. **aliasy neodpovídaly názvům ve stagingu**
2. některé týmy **neexistovaly ve staging tabulce**
3. join používal pouze alias → name match

# 4. Test alias → API match

Diagnostický dotaz:

```sql
select
    a.team_id,
    t.name,
    a.alias,
    s.team_id,
    s.name,
    s.logo
from public.team_aliases a
join public.teams t
  on t.id = a.team_id
left join staging.api_football_teams s
  on lower(trim(a.alias)) = lower(trim(s.name))
where a.team_id in (44,50,87);
```

Výsledek:

* aliasy existují
* ale **neodpovídají názvům v API**

Například:

```
alias: sp lisbon
api:   Sporting CP
```

---

# 5. Aktualizace log týmů

Použitý update:

```sql
update public.teams t
set logo_url = x.logo
from (
    select distinct
        a.team_id,
        s.logo
    from public.team_aliases a
    join staging.api_football_teams s
      on lower(btrim(a.alias)) = lower(btrim(s.name))
) x
where x.team_id = t.id
and (t.logo_url is null or t.logo_url = '');
```

Výsledek:

| Team        | Logo  |
| ----------- | ----- |
| Watford     | OK    |
| Chelsea     | OK    |
| Man City    | OK    |
| Real Madrid | OK    |
| Sporting    | OK    |
| PSG         | OK    |
| Wrexham     | chybí |

```sql
select team_id,name,logo
from staging.api_football_teams
where lower(name) like '%wrex%'
```

Výsledek:

To znamená:

```
API data nebyla ingestována
```
