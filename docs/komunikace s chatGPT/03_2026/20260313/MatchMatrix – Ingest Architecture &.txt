3. Přehled tabulek OPS
3.1 ops.ingest_targets

Definice co je možné ingestovat.

Obsahuje:

provider

sport

league

season

run_group

enabled

Použití:

definuje rozsah ingestu

Příklad:

api_football | football | league 39 | season 2022 | FOOTBALL_MAINTENANCE
3.2 ops.ingest_planner (NOVÁ TABULKA)

Konkrétní pracovní fronta ingest úloh.

Obsahuje:

provider

sport

entity

league

season

run_group

priority

status

attempts

Statusy:

pending
running
done
error

Příklad:

football | fixtures | league 39 | 2022 | pending

Tato tabulka řídí co se bude právě stahovat.

3.3 ops.scheduler_queue

Fronta úloh pro scheduler.

Použití:

scheduler orchestrace

Scheduler rozhoduje:

kdy job spustit
v jakém pořadí
3.4 ops.jobs

Katalog známých systémových jobů.

Například:

unified_ingest_batch
compute_mmr_ratings
build_match_features

Používá se pro:

registry jobů
3.5 ops.job_runs

Audit běhů jobů.

Ukládá:

start_time
end_time
status
log message

Použití:

monitoring systému
3.6 ops.ingest_runtime_config

Runtime konfigurace ingestu.

Obsahuje například:

FREE / PRO režim
season window
max requests

Použití:

globální konfigurace ingestu
3.7 ops.sports_import_plan

Strategie ingestu na úrovni sportů.

Například:

football priority 1
basketball priority 2

Použití:

priority sportů
3.8 ops.league_import_plan

Strategie ingestu na úrovni lig.

Například:

Premier League priority HIGH
2. Bundesliga MEDIUM

Použití:

priority lig
3.9 ops.api_budget_status

Kontrola API budgetu.

Obsahuje:

requests_remaining
requests_used_today

Použití:

ochrana před překročením API limitů
3.10 ops.api_request_log

Log všech API requestů.

Použití:

monitoring API usage
