## Datum
2026-03-13

# 1. Stav před dnešní V2 úpravou

Po dokončení předchozí části projektu byla ověřena funkční ingest pipeline V1:

```text
ops.ingest_targets
→ build_ingest_planner_jobs.py
→ ops.ingest_planner
→ run_ingest_planner_jobs.py
→ ingest/run_unified_ingest_v1.py
→ staging.stg_provider_*
→ run_unified_staging_to_public_merge_v1.py
→ public.*

Bylo potvrzeno, že:

planner builder vytváří planner joby z ops.ingest_targets

planner worker přebírá pending joby a mění je na running

planner worker úspěšně spouští unified ingest runner

data se zapisují do unified staging tabulek

merge worker propisuje data do public.*

vznikl první orchestration wrapper run_ingest_cycle_v1.py

2. Důvod V2

V1 orchestrátor byl funkční, ale chyběly dvě důležité věci:

audit celého cycle běhu do ops.job_runs

ochrana proti souběžnému spuštění více orchestrátorů současně

Bez těchto prvků hrozilo:

duplicitní ingestování

obtížnější audit celého cyklu

nejasný provozní stav při paralelním spuštění

3. Co bylo navrženo ve V2
3.1 Worker lock vrstva

Byla navržena nová tabulka:

ops.worker_locks

Účel:

bránit souběžnému běhu stejného orchestration workeru

držet informaci o ownerovi locku

mít TTL přes expires_at

umožnit heartbeat během delšího běhu

Navržené klíčové sloupce:

lock_name

owner_id

acquired_at

expires_at

heartbeat_at

note

3.2 Orchestrátor V2

Byl navržen nový skript:

C:\MatchMatrix-platform\workers\run_ingest_cycle_v2.py

Jeho funkce:

získá lock ingest_cycle_v2

zapíše audit běhu do ops.job_runs

spustí planner worker

vyhodnotí počet zpracovaných planner jobů

merge spustí pouze tehdy, když planner opravdu něco zpracoval

po dokončení zapíše finální stav do ops.job_runs

uvolní lock

3.3 Audit do ops.job_runs

V2 cycle zapisuje samostatný auditní řádek do:

ops.job_runs

s novým job_code:

ingest_cycle_v2

Tím vzniká oddělený audit pro:

jednotlivé planner joby

celý orchestration cycle

To je důležité pro budoucí panel, monitoring i troubleshooting.

4. Připravené soubory
SQL

C:\MatchMatrix-platform\db\036_ops_create_worker_locks.sql

C:\MatchMatrix-platform\db\037_ops_insert_job_ingest_cycle_v2.sql

Python

C:\MatchMatrix-platform\workers\run_ingest_cycle_v2.py

5. Přínos V2

V2 posouvá MatchMatrix z prostého wrapperu na reálnější orchestration vrstvu:

cycle má vlastní audit

cycle má vlastní lock

merge se spouští jen při skutečné práci

běh je bezpečnější proti souběhu

6. Doporučený další krok

Po nasazení V2:

spustit SQL pro ops.worker_locks

zaregistrovat ingest_cycle_v2 do ops.jobs

otestovat:

běžný běh

běh bez práce

běh se --skip-merge

dvojí spuštění pro ověření locku

7. Další směr po V2

Po ověření V2 bude vhodné pokračovat:

panel nad ops.ingest_planner

panel nad ops.job_runs

čištění / retry politika pro error planner joby

samostatný scheduler loop

budoucí napojení na API budget management

## Datum

# 1. Cíl úpravy

Doplnit samostatný audit pro merge worker, aby byl merge krok viditelný v:

- `ops.job_runs`
- OPS dashboard views

---

# 2. Problém před úpravou

Merge worker:

- `run_unified_staging_to_public_merge_v1.py`

sice běžel správně a propisoval data ze `staging` do `public`,
ale nevytvářel samostatný záznam v `ops.job_runs`.

Důsledek:

- v dashboardu nebylo možné zobrazit poslední merge běh
- merge nebyl odděleně auditovatelný
- troubleshooting mezi staging a public byl horší

---

# 3. Provedená změna

Byl přidán nový job do:

- `ops.jobs`

s kódem:

- `unified_staging_to_public_merge`

A merge worker byl upraven tak, aby:

1. při startu vytvořil řádek v `ops.job_runs`
2. při úspěchu zapsal:
   - `status = ok`
   - `message`
   - `details`
   - `rows_affected`
3. při chybě zapsal:
   - `status = error`
   - text chyby
   - částečné statistiky

---

# 4. Přínos

Po této úpravě bude v OPS vrstvě samostatně viditelné:

- poslední merge běhy
- status merge
- délka merge
- souhrn změněných řádků
- finální počty v `public.*`

To zlepšuje provozní přehled i budoucí monitoring.

---

# 5. Soubory

## Python
- `C:\MatchMatrix-platform\workers\run_unified_staging_to_public_merge_v1.py`

# 6. Další krok
