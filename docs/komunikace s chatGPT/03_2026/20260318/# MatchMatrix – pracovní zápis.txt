# 2. Players ingest pipeline

Pipeline byla spuštěna přes:

```
run_ingest_cycle_v3.py
```

s parametry:

```
provider = api_football
sport = football
entity = players
limit = 20
```

Planner postupně zpracoval **11 jobů**.

### Příklad jobu

```
league=205 season=2024
inserted=60 players
```

Data byla uložena do:

```
staging.players_import
```

a následně merge do:

```
public.players
public.player_provider_map
```

# 3. Stav hráčských dat

Kontrola tabulek:

| tabulka                                  | počet |
| ---------------------------------------- | ----- |
| staging.players_import                   | 820   |
| staging.stg_provider_players             | 533   |
| staging.stg_provider_player_season_stats | 1692  |
| staging.stg_provider_player_stats        | 0     |

### Interpretace

**players_import**

* obsahuje surové hráče z API
* jeden hráč se může objevit vícekrát (různé sezóny)

**stg_provider_players**

* deduplikovaní hráči
* unikátní identity hráčů

**player_season_stats**

* statistiky hráče za sezónu
* více řádků na hráče

**player_stats**

* zatím prázdné
* tato tabulka bude plněna z endpointu:

```
/fixtures/players
```
